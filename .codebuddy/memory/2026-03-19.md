### 发现并修复的 Bug

1. **Domain 前缀重复导致 500（测试循环中发现）**
   - 根因：测试脚本 `rnd('D').toUpperCase().slice(0,5)` 只截取 5 个字符，而 UID 在同一进程内不变，导致每轮生成的前缀完全相同（如 `DMMW9`），触发 `UNIQUE constraint failed: domains.project_id, domains.task_prefix`
   - 修复：改为 `D${ROUND}${Math.random()...}` 确保每轮前缀唯一

2. **Domain 创建路由缺少 try/catch（500 而非 409）**
   - 根因：`routes.ts` 的 `POST /api/v1/domains` 没有 try/catch，UNIQUE 约束失败直接返回 500
   - 修复：添加 try/catch，UNIQUE 错误返回 409，其他返回 500

## 修复：登录后页面空白（Web UI 打不开）

### 问题
用户反映登录后页面无法加载——浏览器显示空白页。

### 根因
`server/src/index.ts` 中注册 `@fastify/static` 时使用了 `wildcard: false`：

```js
await app.register(staticFiles, {
  root: config.webDistPath,
  prefix: '/',
  wildcard: false,  // ← 这行导致 /assets/* 子目录文件无法被自动路由
});
```

`wildcard: false` 导致 `@fastify/static` **不注册 `/*` 通配符路由**，只注册精确的根级文件路由。
因此 `/assets/index-xxx.js`、`/assets/index-xxx.css` 等请求全部匹配不到静态文件路由，
走了 SPA fallback `setNotFoundHandler` → 返回 `index.html`（465字节）。
浏览器请求 JS 文件得到的是 HTML，自然无法执行，页面空白。

### 修复
- `server/src/index.ts`：移除 `wildcard: false`，恢复默认的 `wildcard: true`
- `web/src/components/CreateTaskModal.tsx`：移除不存在的 `form.assignee` 引用（TS 编译错误）
- 重新 `vite build` 前端

## 修复：登录后页面闪屏（闪一下登录页再跳回）

### 问题
用户反映登录后页面反复闪烁——在登录页和主页之间来回跳。

### 根因
三处代码共同导致了闪屏循环：

1. **`App.tsx` 的 `OnboardingGuard`**：只做同步检查 `isAuthenticated` 和 `currentUser`，
   当有 token 但 `currentUser` 为 null 时（页面刷新后 `clawpm-member` localStorage 可能丢失），
   立即 `<Navigate to="/onboarding" />`，不等 `authMe` 查询恢复会话。

2. **`Onboarding.tsx`**：在渲染时如果有 token 会发 `authMe` 请求，但请求返回前显示的是登录表单，
   返回后如果 `currentMember` 存在就 navigate 回 `/my/dashboard`——用户看到登录页闪了一帧。

3. **`Layout.tsx` 的 stale member effect**：如果 `members` 列表中找不到 `currentUser`，
   直接 `clearCurrentUser()` + `clearOnboarded()` → 触发 guard 再跳 onboarding → 死循环。

### 修复
- **`App.tsx`**：`OnboardingGuard` 改为异步 guard。有 token 但无 `currentUser` 时先查 `authMe`，
  查询期间显示 "正在恢复会话" 加载页；查询成功自动恢复 member；查询失败才跳 onboarding。
- **`Onboarding.tsx`**：增加 `authMeLoading` 判断，有 token 且首次加载中时显示加载占位，
  不闪登录表单。
- **`Layout.tsx`**：stale member 时不再 `clearCurrentUser()`，改为弹出身份选择器，
  避免和 guard 形成循环。
