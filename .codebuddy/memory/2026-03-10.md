# 2026-03-10 工作日志

## v3.4 图片上传 + 节点样式增强

### 图片上传功能
- **后端**：在 `server/src/index.ts` 注册了 `@fastify/multipart` 插件和 `/uploads/` 静态文件服务
- **后端**：在 `server/src/api/routes.ts` 新增 `POST /api/v1/upload/image` 路由，支持最大 10MB 的 png/jpeg/gif/webp 图片上传
- **前端**：在 `web/src/api/client.ts` 新增 `uploadImage(file)` API 方法（使用 FormData）
- **前端**：在 `web/src/pages/TaskDetail.tsx` 描述编辑器新增：
  - 粘贴截图自动上传（onPaste 事件检测 clipboardData.files）
  - 拖拽图片到编辑区上传（onDrop + onDragOver）
  - 工具栏"插入图片"按钮（隐藏的 file input）
  - 上传后自动在光标位置插入 `![filename](url)` Markdown 语法
- **依赖**：新增 `@fastify/multipart`，`@fastify/static` 已有

### 脑图节点样式增强
- **数据模型**：`NodeStyle` 接口新增 `borderMode?: 'bar' | 'half' | 'full'` 字段
  - `bar`（默认）：仅左侧色条 + 灰色细边框
  - `half`：半包围（左+上+下三边使用强调色，右侧浅灰）
  - `full`：全包围（四边都用强调色边框）
- **NodeStyleModal**：新增"包围模式"三选一选择器，实时预览三种效果
- **TaskNode**：根据 `borderMode` 计算不同的 `borderStyles` 和 `showBar` 变量
- **持久化**：通过 localStorage nodeStyles，向后兼容（旧数据无 borderMode 字段自动视为 'bar'）

### 个人空间思维导图修复（MyTasks.tsx）
- **发现**：项目有两套独立的思维导图实现：
  - `/mindmap` → `MindMap.tsx`（项目空间）
  - `/my/tasks/mindmap` → `MyTasks.tsx` 中的 `MyTasksMindMapView`（个人空间）
- **折叠/展开按钮修复**：原来 `MmTaskNode` 只在折叠时显示子节点数量小圆圈，没有可点击的展开/折叠按钮。新增了 `▼`/`▶` 按钮（带 `nopan nodrag nowheel` class），通过 `onToggleCollapse` 回调传递到节点
- **节点拖动修复**：原来 `nodesDraggable: true` + `onNodesChange` 导致拖动节点后位置不会复原。改为 `nodesDraggable: false` + 移除 `onNodesChange/onEdgesChange`，节点位置完全由算法布局控制
- **交互调整**：单击→打开详情，双击→折叠/展开（原来单击不操作、双击跳转详情）
- **恢复 MindMap.tsx 调试改动**：移除了之前添加的 DEBUG 日志和"总是显示按钮"的临时代码
